## 一、信息架构

### 1.1 系统概述
营销管理系统是一个面向金融营销业务的综合管理平台，主要服务于代运营业务、营销分评估、账单管理和系统监控等核心场景。

### 1.2 一级模块结构

| 模块名称 | 路由标识 | 业务定位 | 访问方式 |
|---------|---------|---------|---------|
| 首页 | `/index` | 工作台入口 | 主导航栏 |
| 代运营 | `/market` | 营销投放核心业务 | 主导航栏 |
| 营销分 | `/score` | 评分模型与产品管理 | 主导航栏 |
| 账单管理 | `/finance` | 计费与财务对账 | 主导航栏 |
| 监控中心 | `/monitor` | 业务运行监控 | 更多菜单 |
| 工具模块 | `/marketTool` | 辅助工具集 | 更多菜单 |
| 系统管理 | `/system` | 权限与配置管理 | 更多菜单 |

## 五、交互流程

### 5.1 投放策略创建流程

```
1. 选择产品
   └→ 从树形结构中选择目标产品（如：速牛-省呗）

2. 查看产品信息
   └→ 显示批次、撞库率、撞库时间、QPS等信息

3. 新增策略
   └→ 点击"新增策略"按钮
   
4. 配置圈选规则
   ├→ 选择底包
   ├→ 配置前筛规则
   ├→ 选择模型组合
   ├→ 设置模型分区
   └→ 选择运营商

5. 配置运营规则
   ├→ 选择触达方式
   ├→ 配置模板编码
   ├→ 设置策略编码
   ├→ 配置链接编码
   └→ 设置营销分

6. 配置去重规则
   └→ 设置去重配置

7. 保存策略
   └→ 策略进入"未启用"状态

8. 启用策略
   └→ 点击"开启"按钮，策略变为"已启用"

9. 计划控制
   ├→ 开始：启动投放计划
   ├→ 暂停：暂停执行
   ├→ 结束：终止计划
   ├→ 推送：推送人群包
   └→ 重置：重置计划状态
```

### 5.2 计划执行状态流转

```
计划已准备 → 开始 → 计划运行中
              ↓
            暂停 → 计划已暂停 → 继续 → 计划运行中
              ↓
            结束 → 计划已结束
```

### 5.3 账单生成流程

```
1. 单价配置
   ├→ 客户侧单价录入（收入侧）
   └→ 渠道侧单价录入（成本侧）

2. 数据产生
   └→ 营销触达执行（短信/语音发送）

3. 月账单生成
   ├→ 客户侧月详单（按客户维度）
   ├→ 渠道侧月详单（按渠道维度）
   └→ 营销分月账单（按评分服务维度）

4. 总账单汇总
   ├→ 客户侧总账单
   └→ 渠道侧总账单

5. 对账确认
   └→ 财务审核
```

### 6.2 请求特征

- **认证方式：** Cookie-based Session（从登录接口返回）
- **查询参数：** 
  - `channelType`：渠道类型（1/2/3）
  - `t`：时间戳（用于缓存控制，如：`?t=1772608197002`）
- **响应格式：** JSON
- **字典数据：** 大量使用字典接口（`/dict`、`getTouchThirdDict`等）用于下拉选项

### 6.3 前端技术栈识别

- **框架：** Vue 3（从`__vue_app__`可识别）
- **路由：** Vue Router 4
- **UI库：** Element Plus
- **构建工具：** Vite（从`/version.json`推测）
- **状态管理：** 推测使用Pinia或Vuex
